Namma Mart Panel - User Manual

User Manual

This section provides a complete step-by-step guide to using the Namma Mart Panel application, from initial setup to end shift operations.

1. Initial Setup and Installation

  1. Install Node.js: Download and install Node.js (version 14 or higher) from the official website if not already installed.
  2. Download Project: Obtain the Namma Mart Panel project files and extract them to a folder on your computer.
  3. Open Terminal/Command Prompt: Navigate to the project directory using the terminal or command prompt.
  4. Install Dependencies: Run the command npm install to install all required Node.js packages.
  5. Configure Email Settings: Open server.js and update the email configuration (Gmail SMTP settings) for OTP functionality.
  6. Start the Server: Run node server.js to start the application server.
  7. Access Application: Open a web browser and navigate to http://localhost:3000 to access the login page.

2. Login Process

  1. Open Login Page: Ensure the server is running and open http://localhost:3000 in your browser.
  2. Enter Credentials: Input your assigned username and password in the respective fields.
  3. Submit Login: Click the "Login" button to authenticate.
  4. Access Granted: Upon successful login, you'll be redirected to either the admin dashboard or employee interface based on your role.
  5. Troubleshooting: If login fails, verify credentials or contact the admin for assistance.

3. Admin Dashboard Operations

  1. Access Admin Panel: After login as admin, you'll see the admin dashboard with store status and employee list.
  2. Check Store Status: View current store status (open/closed) and shift information.
  3. Start New Shift: Click "Start Shift", enter the admin password, and confirm to open the store for the day.
  4. Employee Management: Click "Manage Employees" to add, edit, or remove employees.
  5. View Reports: Access "Analysis" or "View Report" to monitor transactions and generate insights.
  6. End Shift Process: When ready to close, click "End Shift" to initiate the closing procedure.

4. Employee Management (Admin Only)

  1. Navigate to Employee Management: From admin dashboard, click "Manage Employees".
  2. Add New Employee:
    1. Click "Add Employee".
    2. Fill in employee details: Name, Username, Email, Password.
    3. Click "Save" to create the employee account.
  3. Edit Employee:
    1. Select an employee from the list.
    2. Click "Edit" and modify the required fields.
    3. Click "Save" to update the information.
  4. Delete Employee:
    1. Select the employee to remove.
    2. Click "Delete" and confirm the action.
    3. Note: Employee data may be preserved in history for auditing.
  5. Assign Roles: Ensure employees have appropriate access levels set.

5. Daily Operations for Employees

Counter Selection

  1. Login as Employee: Use employee credentials to log in.
  2. Select Counter: Choose between Counter 1 or Counter 2 from the available options.
  3. Set Shift Times: Enter shift start and end timestamps.
  4. Pine Lab Value: If Counter 1 is selected, enter the Pine Lab Value (required for Counter 1 only).
  5. Confirm Selection: Click "Submit" to save counter selection and begin shift.

Transaction Entry

  1. Access Transaction Forms: From the employee dashboard, select the type of transaction to enter.
  2. Extra Transactions:
    1. Select "Extra" from the menu.
    2. Enter Item Name, Bill Number, Amount, and Mode of Pay (Cash/Card/UPI/Other).
    3. Click "Submit" to record the transaction.
  3. Delivery Transactions:
    1. Select "Delivery" from the menu.
    2. Enter Bill Number, Amount, and Mode of Pay.
    3. Click "Submit" to record.
  4. Bill Paid Transactions:
    1. Select "Bill Paid" from the menu.
    2. Enter Vendor/Supplier name and Amount Paid.
    3. Click "Submit" to record.
  5. Issue Reporting:
    1. Select "Issue" from the menu.
    2. Enter Bill Number and detailed Issue Description.
    3. Click "Submit" to report the issue.
  6. Retail Credit:
    1. Select "Retail Credit" from the menu.
    2. Enter Phone Number, Amount, and Mode of Pay.
    3. Click "Submit" to record.

6. Reporting and Analysis

  1. Access Analysis: From the main menu, click "Analysis" to view data insights and charts.
  2. View Detailed Reports:
    1. Click "View Report".
    2. Select an employee from the list.
    3. Choose a date for the report.
    4. View transaction data categorized by type (Extra, Delivery, etc.).
  3. Apply Filters: Use date ranges, amounts, or other filters to narrow down report data.
  4. Export Reports: Click "Export to Excel" to download filtered data for external use.
  5. View History: Access edit and deletion history with reasons and timestamps.
  6. Edit Records: From reports, edit counter data with reason (requires admin approval).
  7. Delete Records: Delete records with mandatory reason (tracked in history).

7. End Shift Process

  1. Initiate End Shift: From admin dashboard, click "End Shift" when ready to close the store.
  2. Admin Password Verification: Enter the admin password to proceed.
  3. OTP Generation: An OTP (One-Time Password) will be sent to the admin's email.
  4. Enter OTP: Check email and enter the received OTP in the application.
  5. Confirm End Shift: Click "Confirm" to finalize the shift closure.
  6. Generate Report: The system will generate a comprehensive end shift report with all transactions.
  7. Store Closure: The store status changes to closed, preventing further transactions until next shift start.
  8. Email Notification: End shift report may be emailed to designated recipients.

8. Data Management and Maintenance

  1. Data Storage: All application data is stored in the db.json file.
  2. History Tracking: All edits and deletions are logged with timestamps, reasons, and user information.
  3. Restore Deleted Items: From history, restore accidentally deleted records.
  4. Backup Data: Regularly backup db.json to prevent data loss.
  5. Data Integrity: Ensure accurate data entry to maintain report accuracy.
  6. Password Management: Contact admin for password resets if needed.
Important Notes and Best Practices:
  • Always provide detailed reasons when editing or deleting records for audit purposes.
  • Ensure all transaction data is entered accurately to maintain financial integrity.
  • Regularly check email for OTP during end shift process.
  • Backup the db.json file frequently to prevent data loss.
  • Contact admin immediately for technical issues or credential problems.
  • Log out after completing work to maintain security.
  • Verify counter selection and Pine Lab values before starting transactions.
  • Use appropriate transaction categories to ensure proper reporting.
